# Bases de datos en Notion

Si te atrapó el universo de páginas bonitas, markdown, íconos, portadas y enlaces mágicos que vimos en la entrega anterior, prepárate para subir al siguiente nivel. Porque si Notion es un castillo encantado de productividad, las bases de datos son su sala de máquinas secreta. Tal vez eso fue muy dramático, pero la realidad es que, en mi opinión, **las bases de datos** son definitivamente lo distingue de otras aplicaciones.

En Notion, una base de datos ***es una herramienta para organizar y gestionar información, donde cada elemento es una página con propiedades personalizables, permitiendo múltiples vistas y filtrados. (Fuente: Gemini, 2025).***

Y sí: **pueden dar miedo al principio**, pero promesa de nerd amiga: una vez que las entiendes, no vuelves atrás.

# Pero... ¿bases de datos? ¿como Excel?

---

Algo así, pero con algo de esteroides.

Una **base de datos en Notion** es un bloque especial que permite **organizar, visualizar, filtrar y conectar información** de una manera modular y preciosa.

Puedes imaginarlo como un conjunto de tarjetas o filas con superpoderes, que puedes mirar como tabla, galería, calendario, lista, cronograma o kanban (¡todo desde la misma información!).

Lo bonito: **todo esto sin escribir una sola fórmula complicada** (aunque si te gusta el caos, también hay fórmulas 👀).

# ¿Cómo se ve una base de datos?

---

Cuando creas una base desde cero (botón `+`, luego "Database"), te aparece una tabla minimalista con columnas tipo “Nombre”, “Etiquetas”, “Fecha”, etc. Pero cada fila no es una fila cualquiera: es una **página editable**, como las normales.

<aside>
💡

Es decir: **puedes meter dentro de una fila una página completa**, con texto, imágenes, tareas, y hasta otra base de datos 😮‍💨. Inception total.

</aside>

# ¿Y para qué sirve una base de datos?

---

Te voy a dar algunos ejemplos que yo he usado (y seguro tú también podrías):

- Un **registro de tareas** estilo kanban (tipo Trello, pero más bonito).
- Un **catálogo de libros o pelis** con estatus, calificación, notas y fechas.
- Un **planner académico** donde llevas el control de entregas por materia.
- Un **control de artículos para un blog** con fechas, estado y links.
- Una **lista de estudiantes con retroalimentaciones personalizadas**.

Y si te pones más pro, puedes conectarlas entre sí. Por ejemplo: una base de materias + una de tareas, y haces que se hablen entre ellas. Boom. Base relacional.

# Tipos de vista en bases de datos

---

Uno de los superpoderes más útiles es que **una misma base se puede ver de diferentes formas**, según lo que necesites:


| Vista | ¿Para qué sirve? |
| --- | --- |
| **Tabla** | Para ver todo clarito, estilo hoja de cálculo |
| **Galería** | Perfecto para catálogos visuales (con imágenes, portadas, moodboards, etc.) |
| **Lista** | Minimalista y sin ruido visual |
| **Calendario** | Genial para tareas o eventos con fecha límite |
| **Cronograma** | Ideal para proyectos con entregas o procesos más largos |
| **Kanban (Tablero)** | Tu clásico “Por hacer / En proceso / Terminado” |

> Y puedes tener **varias vistas del mismo contenido** sin duplicar nada. Casi magia negra. Pero legal.
> 

## Campos y propiedades

---

Cada columna en una base de datos es una **propiedad**. Y hay muchas:

- **Texto**: libre.
- **Selección única o múltiple**: como etiquetas.
- **Fecha**: con recordatorio si quieres.
- **Casilla de verificación**: útil para tareas.
- **Número**: obvio.
- **Relación**: conecta dos bases entre sí.
- **Fórmula**: para quienes quieren jugar con funciones lógicas.

# Cómo crear una base de datos en Notion

---

1. En cualquier página, escribe `/table` y elige "Tabla – en línea" o "Tabla – completa".
2. Edita las columnas según lo que necesites (tipo, nombre, formato).
3. Empieza a añadir registros (filas), y si quieres, ábrelos y escribe dentro de ellos como una página normal.
4. Añade vistas distintas según lo que estés gestionando.
5. Filtra, ordena, y juega.

<aside>
Antes de volverte loca creando columnas, hazte esta pregunta:

*“¿Esto me ayuda a tomar una decisión o solo estoy haciendo bonito?”*

## Ejemplo práctico: planificador de tesis 😅

---

Supongamos que estás haciendo una base de datos con:

- **Título del capítulo**
- **Estado (Por escribir, En proceso, Listo)**
- **Fecha estimada de entrega**
- **Notas**
- **Vínculo con tu tutor o asesora (como página relacionada)**

Puedes filtrar solo los capítulos pendientes, crear un calendario con las fechas, y además meter en cada página el texto de cada capítulo. ¡Todo en un mismo lugar!
